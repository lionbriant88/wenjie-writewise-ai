## 下一步最合理开发内容

建议下一步回到上传/OCR 的真实教师使用场景，完善“手动作文分组”。

优先方向：

1. 保留现有“合并为一篇多页作文”和“按图片拆分为多篇作文”两个简单模式。
2. 增加手动分组模式，支持图片 1-2 为一篇、图片 3 为一篇、图片 4-5 为一篇这类混合情况。
3. 让老师在确认 OCR 前预览每个作文组的页数和文本草稿。
4. 手动分组只作为高级选项出现，避免打扰普通上传流程。
5. 为手动分组、OCR 确认、进入批改队列增加聚焦测试和浏览器验证。
