# 文阶 WriteWise AI 第一阶段静态原型开发任务清单

## 0. 项目目录约束

产品中文名为“文阶”，英文名为 “WriteWise AI”。

后续所有开发文件、素材、文档和导出物建议统一放在 D 盘。

推荐项目根目录：

```text
D:\wenjie-writewise-ai
```

第一阶段前端原型建议放在：

```text
D:\wenjie-writewise-ai\app
```

文档建议放在：

```text
D:\wenjie-writewise-ai\docs
```

工程目录使用英文小写，避免中文路径导致依赖安装、脚本运行或部署时出现兼容问题。

## 1. 阶段目标

第一阶段目标不是接入真实 OCR 或真实 AI，而是先做出一个可以演示完整流程的网页原型。

老师看到这个原型后，应能理解产品如何完成：

创建批改任务 → 上传作文 → 整理多页作文 → 后台批改进度 → 异常复核 → 单篇作文结果 → 班级共性问题讲评

第一阶段使用模拟数据，重点验证产品流程、页面结构和教师使用体验。

## 2. 技术建议

建议第一版使用网页应用实现。

推荐技术栈：

- React
- TypeScript
- Vite
- Tailwind CSS
- 本地 mock 数据

第一阶段不做后端、不做数据库、不做账号登录。

数据可以先写在前端 mock 文件中，后续再替换成真实接口。

## 3. 页面清单

### 3.1 任务列表页

用途：教师查看所有作文批改任务。

需要展示：

- 任务名称
- 班级名称
- 作文类型
- 作文数量
- 已完成数量
- 异常数量
- 当前状态
- 创建时间

需要操作：

- 创建新任务
- 进入任务详情
- 查看班级讲评

### 3.2 创建任务页

用途：教师创建一次作文批改任务。

字段：

- 任务名称
- 班级名称
- 作文类型
- 满分，默认 15 分
- 评分模板
- 是否生成班级讲评

评分维度默认展示：

- 卷面/字迹
- 内容完成度
- 语言准确性
- 表达清晰度
- 词汇句式
- 意图表达清晰度

第一阶段可以只做表单和保存模拟任务，不需要真实持久化。

### 3.3 上传作文页

用途：教师批量上传作文图片。

第一阶段使用模拟上传，不需要真的识别图片。

需要展示：

- 上传区域
- 已上传图片列表
- 每张图片的预览占位
- 图片所属作文编号
- 是否为多页作文

需要操作：

- 添加图片
- 将多张图片合并为一篇作文
- 拆分误合并的页面
- 调整页序
- 提交到批改队列

### 3.4 批改进度页

用途：教师看到系统正在后台批量处理作文，而不是一篇篇等待。

需要展示：

- 总作文数
- 已完成数
- 异常数
- 当前处理进度条
- 每篇作文状态

作文状态包括：

- 待识别
- 识别中
- 待批改
- 批改中
- 已完成
- 需人工复核

需要操作：

- 查看异常作文
- 查看已完成作文
- 查看班级讲评

### 3.5 异常复核页

用途：教师只处理系统无法可靠处理的作文。

异常类型：

- OCR 置信度低
- 图片模糊
- 字迹太乱

需要展示：

- 作文编号
- 异常原因
- 原图预览
- OCR 识别文本
- OCR 置信度
- 多页缩略图

需要操作：

- 修改 OCR 文本
- 调整页序
- 合并页面
- 拆分页面
- 重新批改
- 标记为人工批改

### 3.6 单篇作文结果页

用途：教师查看某篇作文的 AI 批改结果。

需要展示：

- 作文编号
- 原图预览
- OCR 文本
- 总分，满分 15 分
- 分项评分
- 全文错误标注
- 错句修改建议
- 升级表达
- 总评

需要操作：

- 修改总分
- 修改分项分
- 修改错误标注
- 修改修改建议
- 保存教师调整

### 3.7 班级讲评页

用途：白板端或电脑端直接展示本次作文的共性问题。

需要展示：

- 高频语法错误
- 高频拼写错误
- 典型问题句
- 可上课讲的改写练习

白板展示要求：

- 字体足够大
- 一屏不要放太多信息
- 页面适合横屏大屏展示
- 不依赖学生姓名

## 4. 模拟数据要求

第一阶段至少准备：

- 3 个批改任务
- 每个任务 8-12 篇作文
- 至少 2 篇异常作文
- 至少 2 篇多页作文
- 至少 5 条语法错误
- 至少 5 条拼写错误
- 至少 5 个典型问题句
- 至少 5 个错句修改建议
- 至少 5 个升级表达

## 5. 核心组件建议

可以拆成以下组件：

- AppLayout
- Sidebar
- TaskCard
- TaskStatusBadge
- UploadPanel
- EssayPageSorter
- ProgressSummary
- ExceptionList
- EssayImagePreview
- OcrTextEditor
- ScoreBreakdown
- ErrorAnnotationList
- RevisionSuggestionList
- ClassInsightPanel

## 6. 路由建议

建议第一阶段路由如下：

- `/`：任务列表
- `/tasks/new`：创建任务
- `/tasks/:taskId/upload`：上传作文
- `/tasks/:taskId/progress`：批改进度
- `/tasks/:taskId/exceptions`：异常复核
- `/tasks/:taskId/essays/:essayId`：单篇作文结果
- `/tasks/:taskId/class-review`：班级讲评

## 7. 第一阶段完成标准

第一阶段完成时，应满足：

- 可以从任务列表进入完整流程
- 可以创建一个模拟批改任务
- 可以看到批量作文上传界面
- 可以看到多页作文整理界面
- 可以看到后台批改进度
- 可以看到异常复核列表
- 可以查看单篇作文批改报告
- 可以修改模拟分数和建议
- 可以查看班级讲评页
- 页面在电脑端和白板横屏上都清晰可读
