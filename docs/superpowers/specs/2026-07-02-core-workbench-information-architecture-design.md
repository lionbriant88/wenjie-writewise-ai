# 核心批改工作台信息架构与信息密度优化设计规格

## Goal

把单篇详情页和班级总览页从“长页面报告”优化为更高效的教师工作台。通过页面内部 Tabs 拆分教师任务、删除冗余视觉大块、压缩卡片信息密度，降低老师在 mock 闭环演示中的滚动和查找成本。

## Current Baseline

当前产品已经形成较完整的 mock 闭环：

```text
上传整理 -> OCR -> 批改进度 -> 单篇详情页 -> 加入班级总览 -> 班级总览展示讲评素材
```

现状问题不在功能缺失，而在两个核心页面开始变重：

- 班级总览页把分数分布、教师精选素材、高频语法错误、高频拼写错误、典型问题句、可上课改写练习全部堆在一个长页面里。
- 班级总览页顶部黑色大横幅与 `AppLayout` 标题区和左侧任务卡片重复，占用首屏。
- 单篇详情页右侧把评分诊断、问题批改、全文优化、教师反馈纵向堆叠，真实作文问题增加后会明显拉长页面。
- 两个页面都已经是“工作台”，下一步应优化任务分区，而不是继续加内容。

## Product Decision

采用“轻量工作台 Tabs”方案：

1. 班级总览页删除黑色大横幅。
2. 班级总览页新增内部 Tabs：
   - 概览
   - 教师精选素材
   - 高频问题
   - 改写练习
3. 单篇详情页保留左侧学生作文原文常驻。
4. 单篇详情页右侧新增内部 Tabs：
   - 评分诊断
   - 问题批改
   - 全文优化
   - 教师反馈
5. 两个页面的 Tabs 都只使用本地 state，不新增路由，不改变现有 URL。
6. 保留现有业务能力、数据结构和 mock 闭环。

本轮只做信息架构和密度优化，不新增复杂功能。

## Non-Goals

本轮明确不做：

- 不改上传整理页。
- 不改批改进度页核心逻辑。
- 不接真实 AI。
- 不接真实 OCR。
- 不接后端数据库。
- 不做账号权限。
- 不做 Word / PDF 导出。
- 不做课堂播放模式。
- 不做 OCR 原文行内批注 v0.1。
- 不做原卷图片批注。
- 不做 OCR 坐标级标注。
- 不大改全局导航。
- 不改变 `classReviewMaterials` 数据结构。
- 不新增表达提升加入班级总览能力。
- 不做详情页 query param 自动定位。

## Class Review Page Design

### Remove Hero Banner

删除当前黑色大横幅：

```text
九年级 3 班
九年级建议信单元测
先看全班分数结构，再看高频问题和课堂讲评素材。
```

理由：

- 与页面顶部标题区重复。
- 与左侧任务信息重复。
- 占用首屏。
- 与当前轻科技教师工作台风格不够统一。

`AppLayout` 的标题和描述保留，并建议描述改为：

```text
先看全班表现，再整理课堂讲评素材。
```

### Internal Tabs

班级总览内容顶部增加内部 Tabs：

- `概览`
- `教师精选素材`
- `高频问题`
- `改写练习`

默认 Tab：`概览`。

Tabs 要求：

- 只影响 `/tasks/:taskId/class-review` 页面内部内容。
- 不新增新路由。
- 侧边栏“班级总览”继续高亮。
- 使用 `useState` 管理 active tab。
- 样式与现有进度页/全文优化稿 Tabs 保持一致：安静、清晰、不过度装饰。

### Tab: 概览

放置：

- 作文总数
- 平均分
- 最高分
- 最低分
- 分数分布条形图
- `1-3`、`4-6`、`7-9`、`10-12`、`13-15` 分数段

要求：

- 保留现有分数统计逻辑。
- 不新增重复统计区域。
- 可小幅压缩外层 padding。
- 该 Tab 的目标是一屏内判断全班整体表现。

### Tab: 教师精选素材

放置当前 `ClassReviewMaterialsPanel`。

保留能力：

- 素材总数。
- 内部筛选：全部、典型错误、逻辑问题。
- `expression_upgrade` 类型已有数据时再显示表达提升 Tab；没有真实素材时不显示空 Tab。
- 素材卡片。
- 查看来源。
- 移除。
- 空状态。

密度优化：

- 素材卡片保持紧凑，不使用大面积灰底卡片。
- 标签、影响等级、建议教师复核和来源作文集中在顶部一行。
- 原句、改法、诊断、讲评建议使用紧凑 label + 内容。
- 按钮放底部，但 padding 保持克制。

### Tab: 高频问题

整合：

- 高频语法错误
- 高频拼写错误
- 典型问题句

本轮不继续使用当前大卡片版 `ClassInsightPanel large`。新增或调整为紧凑列表展示。

每组结构：

```text
高频语法错误 2
suggest 后动词形式错误        8 次
学生常把 suggest you do 写成 suggest you does / joins.
例句：I suggest you joins the club.
```

要求：

- 每组标题显示数量。
- 每条默认最多 2-3 行。
- 不做展开/收起；长文本通过紧凑排版和自然换行处理。
- 高频语法错误、高频拼写错误、典型问题句原有数据不丢失。
- 空状态：`当前暂无高频问题。`

### Tab: 改写练习

放置当前 `rewriteExercises` 数据。

推荐展示结构：

```text
把笼统评价改具体
原句：It is very good for you.
改写：It will give you a valuable chance to practice English.
说明：表达更具体，适合建议类作文。
```

要求：

- 使用紧凑列表。
- 不做课堂播放模式。
- 不做大卡片嵌套。
- 原有可上课改写练习数据不丢失。
- 空状态：`当前暂无可上课改写练习。`

## Essay Detail Page Design

### Preserve Left Source Panel

左侧继续常驻：

- 学生作文原文。
- OCR 置信度。
- 查看原图。
- 阅读定位。
- 编辑 OCR。

不在左侧加 Tab，不隐藏左侧，不改变现有定位/编辑能力。

### Right-Side Internal Tabs

右侧内容区顶部新增内部 Tabs：

- `评分诊断`
- `问题批改`
- `全文优化`
- `教师反馈`

默认 Tab：`评分诊断`。

要求：

- Tab 切换只影响右侧内容。
- 不新增新路由。
- 不破坏 `/tasks/:taskId/essays/:essayId`。
- 不破坏返回批改进度、上一篇、下一篇。
- 当前顶部保存反馈 `saveNotice` 可以继续放在右侧工作区顶部，显示在 Tabs 上方或下方均可；优先保持现有交互反馈清楚。

### Tab: 评分诊断

放置：

- `DiagnosticScoreSummary`
- 总分。
- AI 置信度。
- 档次。
- 讲评建议。
- 主要扣分项。
- 分项评分编辑。

要求：

- 分项评分仍可编辑。
- 总分仍随分项评分联动。
- `teacherAdjusted` 和保存反馈状态不破坏。
- 目标是老师一屏内判断分数是否合理。

### Tab: 问题批改

放置：

- `IssueCorrectionList`
- 语言问题。
- 逻辑问题。
- 加入班级总览。
- 已加入班级总览状态。
- 问题定位状态。

保留能力：

- 点击问题卡片后，左侧原文仍高亮定位。
- 逻辑问题仍显示“建议教师复核”。
- 加入班级总览仍写入全局素材池。
- 已加入状态仍从全局素材池派生。
- 从班级总览移除素材后，回到详情页按钮仍恢复为“加入班级总览”。

本轮不新增二级筛选。`全部 / 语言问题 / 逻辑问题 / 已加入总览` 可作为后续优化。

密度优化：

- 问题卡片继续使用顶部标签 + 右上按钮结构。
- 减少不必要说明文字。
- 原句、改法、原因、诊断、建议处理使用紧凑 label + 内容。
- 不牺牲可读性，不做过度压缩。

### Tab: 全文优化

放置：

- `FullTextRevisionPanel`
- 纠错版。
- 提升版。
- 逐句对照。
- 逻辑优化说明。
- 本文重点提升点。

要求：

- 保留 `FullTextRevisionPanel` 自身内部 Tabs。
- 默认仍由该组件控制，当前为“提升版”。
- 不新增表达提升加入班级总览逻辑。
- 不大改全文优化稿内部结构。

### Tab: 教师反馈

放置：

- AI 总评。
- 教师补充建议。
- 保存调整按钮。

要求：

- AI 总评仍可编辑。
- 教师补充建议仍可编辑。
- 保存调整仍显示成功反馈。
- 不破坏 `teacherAdjusted`。
- 不影响评分编辑保存逻辑。

### Top Review Action Bar

本轮不主动重构顶部详情导航。

保留：

- 返回批改进度。
- 作文编号。
- 总分和 AI 置信度。
- 上一篇 / 下一篇。

如果实施时出现明显重复，可做轻微文案压缩，但不作为主要任务。

## Shared Density Guidelines

两个页面统一遵守：

- 不使用 page section 级大卡片嵌套。
- 卡片圆角保持 8px 左右。
- 标题字号与容器大小匹配，避免紧凑面板内过大标题。
- 标签和来源信息尽量合并到一行。
- 不为了“丰富”堆说明文案。
- 视觉上保持简洁、专业、轻科技感。
- 不使用营销页式 hero、渐变装饰、浮夸视觉。

## Acceptance Criteria

- 两个核心页面不再把所有内容纵向堆在同一个长滚动流里。
- 班级总览页删除冗余黑色大横幅。
- 班级总览页内部 Tabs 能稳定切分概览、教师精选素材、高频问题、改写练习。
- 单篇详情页右侧 Tabs 能稳定切分评分诊断、问题批改、全文优化、教师反馈。
- 左侧作文原文阅读/编辑/定位能力不被破坏。
- 班级总览素材池闭环不被破坏。
- `classInsights` 原有数据不丢失。
- 全文优化稿和教师反馈能力不被破坏。
- 批改进度页最新完成入口不被破坏。
